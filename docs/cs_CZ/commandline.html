Parametry příkazového řádku

Při spouštění lze pgAdminu zadat několik různých voleb příkazového řádku.

  • -h Zobrazí nápovědu k volbám příkazového řádku.
     
  • -s <název serveru> Automaticky se připojit k zadanému serveru. <název serveru> je název uvedený při přidávání nového serveru v pgAdminu. Pokud název obsahuje mezery nebo speciální znaky, je zapotřebí jej uzavřít do uvozovek. Pokud má více serverů stejný název, použije se první z nich.
     
  • -S Otevřít okno s informacemi o stavu serveru. Pokud se použije spolu s volbou -s <název serveru>, tak se automaticky připojí k danému serveru. Pokud se použije samostatně, objeví se dialog s dotazem na připojovací údaje. Tuto volbu nelze kombinovat s volbou -Sc <připojovací řetězec>.
     
  • -Sc <připojovací řetězec> Otevřít okno s informacemi o stavu serveru. Při spuštění se automaticky připojí za použití zadaného připojovacího řetězce. Tuto volbu nelze kombinovat s volbou -S.
     
  • -q Otevřít okno s dotazovacím nástrojem. Pokud se použije spolu s volbou -s <název serveru>, tak se automaticky připojí k danému serveru. Pokud se použije samostatně, objeví se dialog s dotazem na připojovací údaje. Tuto volbu nelze kombinovat s volbou -qc <připojovací řetězec>.
     
  • -qc <připojovací řetězec> Otevřít okno s dotazovacím nástrojem. Při spuštění se automaticky připojí za použití zadaného připojovacího řetězce. Tuto volbu nelze kombinovat s volbou -q.
     
  • -f <SQL skript> Automaticky otevřít zadaný SQL skript v dotazovacím nástroji. Lze použít pouze s volbou -q nebo -qc.
     
  • -ch <soubor hba> Upravit zadaný soubor pg_hba.conf s nastavením.
     
  • -cm <soubor nastavení> Upravit zadaný soubor postgresql.conf s nastavením.
     
  • -cp <soubor pgpass> Upravit zadaný soubor .pgpass/pgpass.conf s nastavením.
     
  • -c <složka> Upravit soubor s nastavením v zadané složce clusteru.
     
  • -t Spustit dialog pro testování překladů. V tomto režimu, který napomáhá kontrole při překládání, lze zobrazit každý dialog bez spouštění celé aplikace.
