Étapes de pgAgent

Chaque job consiste en un certain nombre d'étapes, chacune d'entre elle pouvant être un script SQL ou un script shell/batch suivant le système d'exploitation. Chaque étape dans un job donné est exécuté à son tour dans un ordre alphanumérique.

Les étapes peuvent être ajoutées à un job via le dialogue de propriétés du job ou en tant que sous-objet. L'onglet Propriétés de la fenêtre principale de pgAdmin affichera les détails de l'étape sélectionnée, et l'onglet Statistiques affichera les détails de chaque exécution de l'étape, incluant sortie et erreurs du script.

Chaque étape consiste en les détails affichées sur la copie d'écran ci-dessous, la plupart s'expliquant d'eux-même. Si Genre est SQL, alors il va sans dire qu'une base de données doit être sélectionnée pour exécuter le script. S'il est configuré à Batch, la chaîne de connexion à la base doit être laissée vide. L'option En cas d'erreur contrôle comment les échecs de cette étape affecteront le statut du job complet.

L'onglet Définition contient un simple champ texte dans lequel le script de l'étape doit être saisir. Pour les étapes SQL, cela doit correspondre à une série d'une ou plusieurs instructions SQL. Pour les jobs batch, sur un serveur WIndows, le syntaxe du fichier batch standard doit être utilisé alors que pour un serveur *nix, tout script shell peut être utilisé, à condition que l'interpréteur adéquat soit indiqué en première ligne (par exemple #!/bin/sh).
