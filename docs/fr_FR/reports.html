Outil de création de rapports

pgAdmin inclut un simple moteur de création de rapports qui vous permet de générer rapidement des rapports à partir des requêts écrites dans l'outil de requêtage ou à partir des objets ou collections d'objets affichés dans la fenêtre principale de l'application. Par exemple, vous pouvez créer un rapport des propriétés de tout objet ou une liste des fonctions d'un schéma. Pour créer un rapport, sélectionnez un nœud dans le navigateur et sélectionnez le rapport à créer à partir du menu contextuel ou à partir du sous-menu Rapports du menu Outils. Pour créer un rapport à partir de l'outil de requêtage, sélectionnez l'option Rapport rapide dans le menu Fichier.

Chaque rapport contient un titre et des notes supplémentaires pouvant être modifiés avant la création du rapport. De plus, vous pouvez sélectionner d'inclure ou non les requêtes SQL nécessaires à la création du rapport.

Les rapports sont générés en interne en XML. Néanmoins, pgAdmin peut appliquer une feuille de style XML au rapport lors de la création pour formatter la sortie de la façon souhaitée. Une feuille de style est fournie par défaut pour réaliser un export HTML. Avec cette option sélectionnée, vous pouvez demander l'intégration de la feuille de style CSS qui donnera au rapport les mêmes couleurs que celles du site pgAdmin), ou l'intégration d'une feuille de style externe ou l'ajout d'un lien vers une feuille de style externe. Les identifiants de classe/objet suivant sont utilisés :

  • #ReportHeader: Ce div contient l'en-tête du rapport.
  • #ReportNotes: Ce div contient les notes optionnelles du rapport.
  • #ReportDetails: Ce div contient le corps principal du rapport.
  • #ReportFooter: Ce div contient le bas de page du rapport.
  • .ReportSQL: Cette classe est utilisée par les blocs <PRE></PRE> contenant le SQL.
  • .ReportDetailsOddDataRow: Cette classe est appliquée aux lignes impaires des tables.
  • .ReportDetailsEvenDataRow: Cette classe est appliquée aux lignes paires des tables.
  • .ReportTableHeaderCell: Cette classe est appliquée aux cellules de l'en-tête de la table.
  • .ReportTableValueCell: Cette classe est appliquée aux cellules de données de la table.
  • .ReportTableInfo: Cette classe est appliquée aux cellules de base de page de la table.

Lors de la génération de rapports au format XML, vous pouvez choisir un XML simple, un XML lié à une feuille de style XSL externe ou de traiter le XML en utilisant une feuille de style externe et de sauvegarder le résultat. Ceci donne une grande flexibilité pour formatter des rapports.

La feuille de style XSL par défaut utilisée pour créer la sortie XHTML peut être utilisée comme base de départ pour la votre. Des données XML peuvent être utilisées pour tester.
