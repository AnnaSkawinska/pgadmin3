Erreurs de connexion

Lors de la connexion à un serveur PostgreSQL, vous pouvez obtenir des messages d'erreur nécessitant un peu plus d'explications. Si vous en rencontrez un, merci de lire les informations suivantes avec attention, nous sommes certains qu'elles vous aideront.

  • could not connect to Server: Connection refused
  • FATAL: no pg_hba.conf entry

could not connect to Server: Connection refused

Il y a deux raisons à ce message. Le serveur peut tout simplement ne pas être démarré. Commencez par le lancer.

L'autre possibilité est que le serveur n'est pas configuré pour accepter les requêtes TCP/IP sur l'adresse affichée.

Pour des raisons de sécurité, un serveur PostgreSQL "tout neuf" n'écoute pas sur les ports TCP/IP. En fait, il doit être activé pour écouter. Ceci se fait en ajoutant tcpip = true au fichier postgresql.conf pour les versions 7.3.x et 7.4.x, et listen_addresses='*' pour les versions 8.0.x et ultérieures ; cela permettra au serveur d'accepter les connexions sur n'importe quelle interface IP.

Pour plus d'informations, merci de vous référer à la documentation de PostgreSQL sur la configuration du serveur.


FATAL: no pg_hba.conf entry

Si ce message apparaît, votre serveur est accessible sur le réseau mais n'est pas configuré pour accepter votre connexion. Votre client n'est pas détecté comme un utilisateur légal de la base de données.

Vous devez ajouter une entrée de la forme
   host template1 postgres 192.168.0.0/24 md5
pour IPv4 ou de la forme
   host template1 postgres ::ffff:192.168.0.0/120 md5
pour IPv6 dans le fichier pg_hba.conf.

Pour plus d'informations, merci de vous référer à la documentation de PostgreSQL sur l'authentification des clients.
