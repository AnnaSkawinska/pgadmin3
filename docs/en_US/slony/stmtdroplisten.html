Name

DROP LISTEN — Eliminate configuration indicating how Slony-I node listens for events

Synopsis

DROP LISTEN (options);

Description

Remove a “listen” configuration entry.

ORIGIN = ival

Node ID of the event origin the receiver is listening for.

PROVIDER = ival

Node ID of the node from which the receiver gets events that come from the origin. If not specified, default is the origin.

RECEIVER = ival

The ID of the node receiving the events.

This uses schemadocdroplisten( integer, integer, integer ).

Example

     DROP LISTEN ( ORIGIN = 1, RECEIVER = 2, PROVIDER = 3 );

Version Information

This command was introduced in Slony-I 1.0. As of version 1.1, you should not need to use it anymore.
