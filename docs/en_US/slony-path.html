Creating paths and listens

Create paths to other nodes


Slony-I needs path information, that defines how a slon process can communicate to other nodes. The conninfo string takes a connect string as described in the libpq connection documentation. Usually, you will need to specify host, dbname and username, while the password should be stored in the .pgpass file.

Create listens to other nodes


After the communication path has been defined, the slon processes need to be advised to listen to events from other nodes. This step is not necessary for Slony-I V1.1 and later, because listen information is generated automatically when paths are defined.
